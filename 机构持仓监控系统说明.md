# 机构持仓监控系统 - 使用说明

## 系统概述

这是一个**基于富途API的机构资金流向监控系统**，通过分析超大单、大单、中单、小单的资金流入流出，自动识别：

- ✅ **机构买入** → 推荐跟进
- ⚠️ **散户买入** → 避雷信号

---

## 核心逻辑

### 资金分类（富途API）

富途的 `get_capital_distribution()` API 返回4种资金类型：

| 资金类型 | 单笔金额 | 代表群体 |
|---------|---------|---------|
| 超大单 (super) | > 100万 | 大机构（JP摩根、高盛等） |
| 大单 (big) | 50-100万 | 中型机构、大户 |
| 中单 (mid) | 10-50万 | 散户、小机构 |
| 小单 (small) | < 10万 | 散户 |

### 我们的判断标准

```
机构资金 = 超大单净流入 + 大单净流入
散户资金 = 小单净流入 + 中单净流入

if 机构净流入 > 0:
    → 💎 推荐信号（机构在增持，可跟进）

if 散户净流入 > 0.5亿:
    → ⚠️ 避雷信号（散户追高，谨慎跟进）

if 机构和散户都净流入:
    → ⚡️ 博弈信号（分析资金占比，判断谁占优）
```

---

## 三种信号类型

### 1. 💎 纯推荐信号（机构买入）

**示例**：蔚来汽车
- 机构净流入：0.18亿
- 散户净流入：≤0.5亿（或净流出）

**投资建议**：
- 机构在增持，可跟进
- 建议仓位：10-30%（根据流入金额）
- 止损位：-5%

### 2. ⚠️ 纯避雷信号（散户买入）

**示例**：美团
- 机构净流入：≤0 或很小
- 散户净流入：1.13亿

**风险提示**：
- 散户在追高，谨慎跟进
- 可能成为接盘侠
- 建议观望或轻仓

### 3. ⚡️ 博弈信号（机构+散户都买入）

**示例**：比亚迪
- 机构净流入：1.30亿（43.8%）
- 散户净流入：1.54亿（56.3%）

**分析**：
- 机构占比 < 50% → 散户资金占优，谨慎追高
- 机构占比 > 50% → 机构资金占优，可考虑跟进
- 存在博弈，注意风险，建议轻仓试探

- 每小时检查一次（交易时间）

## 常见问题

### Q1: 为什么没有找到JP摩根的持仓数据？

**A**: 我们不是直接查询JP摩根的持仓，而是通过**资金流向**判断机构行为：
- 超大单+大单 = 机构资金（包括JP摩根、高盛等所有大机构）
- 小单+中单 = 散户资金（包括富途用户等）

这个方法更实时、更准确！

### Q2: 为什么是每小时运行，而不是每分钟？

**A**:
- 资金流向数据相对稳定（不像K线那样波动）
- 每小时检查足够捕捉趋势
- 避免过度交易和API调用限制

### Q3: 如何调整信号灵敏度？

**A**: 编辑 `institutional_monitor.py`：

```python
# 当前阈值
if inflow > 0:          # 机构任何净流入都推荐
if inflow > 0.5:        # 散户净流入 > 0.5亿才避雷

# 调整为更严格（只推荐大额流入）
if inflow > 1.0:        # 机构净流入 > 1亿才推荐
if inflow > 2.0:        # 散户净流入 > 2亿才避雷
```

### Q4: FutuOpenD未运行怎么办？

**A**:
```bash
# 检查FutuOpenD是否运行
ps aux | grep FutuOpenD

# 如果没运行，启动FutuOpenD
# 参考之前的启动脚本
```
