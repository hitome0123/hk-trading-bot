## 如果要演示运行

### 最小配置方案（只需2个凭证）

只配置这2个就能演示基本流程：

#### 1. PostgreSQL（必需）

```bash
# 安装PostgreSQL
brew install postgresql@14
brew services start postgresql@14

# 创建数据库
createdb ecommerce

# 导入schema
psql -d ecommerce -f sql/schema.sql
psql -d ecommerce -f sql/sample-data.sql
```

**在n8n中配置**:
- Settings → Credentials → New → Postgres
- Host: localhost
- Database: ecommerce
- User: postgres
- Password: (留空或你的密码)
- Port: 5432

#### 2. OpenAI API（可选，演示AI功能）

**获取API Key**:
1. 访问 https://platform.openai.com/api-keys
2. 注册并创建API key
3. 复制key（sk-开头）

**在n8n中配置**:
- Settings → Credentials → New → OpenAI
- API Key: sk-your-key

**成本**: 测试一次 < $0.10

### 演示流程（使用最小配置）

```bash
# 1. 打开主workflow
open http://localhost:5678/workflow/PosUVoqAwANQNPlD
```

## 面试常见问题应对

**Q: 为什么有这么多凭证没配置？**
A: "这是一个完整的企业级设计，集成了7个外部服务。作为求职作品，我重点展示架构设计能力。实际上线时，配置这些凭证是标准流程，通常20-30分钟可以完成。"

**Q: 能实际运行吗？**
A: "可以。我准备了两个方案：1) 最小配置（只需PostgreSQL）可以演示核心流程；2) 完整配置需要注册各平台API，我已准备了详细的部署指南。您想看哪个？"

**Q: 生产环境怎么管理这么多凭证？**
A: "企业级实践：1) 使用密钥管理服务（AWS Secrets Manager、Vault）；2) 定期轮换（建议90天）；3) 最小权限原则；4) 审计日志追踪。n8n本身提供了凭证加密存储。"
