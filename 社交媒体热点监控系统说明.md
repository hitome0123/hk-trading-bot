# 社交媒体热点监控系统 - 使用说明

## 系统概述

这是一个**自动化的社交媒体热点监控和港股投资信号系统**，通过以下流程工作：

```
采集社交媒体热点 → 计算K值(热度趋势) → 映射到港股板块 → 推送到Telegram
```

---

## 核心功能

### 1. 多平台数据采集
- **微博热搜** (目前使用模拟数据)
- **抖音热榜** (目前使用模拟数据)
- **淘股吧热帖** (目前使用模拟数据)
- **豆瓣投资小组** (目前使用模拟数据)

### 2. K值趋势分析
- **K值计算公式**: K = (当前热度 - 平均热度) / 平均热度
- **K值判断标准**:
  - K > 1.0 → 爆发式上涨 (可能已到顶)
  - 0.3 < K < 1.0 → 加速上涨 ✅ **理想入场区间**
  - 0 < K < 0.3 → 平稳
  - K < 0 → 下降

### 3. 智能股票映射
关键词自动映射到港股板块：
- 新能源汽车 → HK.09866, HK.02015, HK.01211...
- AI科技 → HK.00700, HK.09988, HK.09618...
- 半导体 → HK.00981, HK.01347, HK.01478...
- 医药生物 → HK.06160, HK.02269, HK.02359...
- 等

### 4. Telegram实时推送
当检测到K值0.3-1.0的上升热点时，自动推送到Telegram

## 自动化配置

### Cron定时任务
系统每10分钟自动执行一次采集→分析→格式化流程：

```bash
*/10 * * * * /Users/mantou/hk-trading-bot/run_social_signals.sh >> /tmp/social_signals_cron.log 2>&1
```

**查看日志**:
```bash
tail -f /tmp/social_signals_cron.log
```

### n8n工作流
工作流配置文件: `/tmp/social_signal_push_workflow.json`

**导入步骤**:
1. 打开n8n: http://localhost:5678
2. 点击右上角"+"创建新工作流
3. 点击右上角菜单 → Import from File
4. 选择 `/tmp/social_signal_push_workflow.json`
5. 点击"Activate"激活工作流

**工作流节点**:
1. **每10分钟检查** - Schedule Trigger
2. **读取消息文件** - Read Binary File
3. **解析JSON** - Code Node
4. **有新信号？** - IF Node
5. **拆分消息** - Code Node
6. **发送到Telegram** - HTTP Request

## 当前数据源说明

⚠️ **重要**: 目前所有平台都使用**模拟数据**，用于演示系统架构和K值计算逻辑。

### 为什么使用模拟数据？
1. 真实API需要企业认证或付费
2. 直接爬虫面临反爬虫限制
3. 先验证系统流程，再对接真实数据

### 如何替换为真实数据？

编辑 `social_media_collector.py`，在每个采集函数中：

```python

    # TODO: 替换为真实API
    import random
    mock_keywords = [...]

# 替换为（真实API）

    url = "https://real-api-endpoint.com/weibo/hot"
    response = requests.get(url, headers=self.headers)
    # 解析真实数据...
```

### 真实数据源选项

**方案A: 付费数据服务**
- 微博/抖音: 新浪数据API、字节跳动开放平台
- 优点: 稳定可靠
- 缺点: 需要付费

**方案B: 开源爬虫**
- 使用Selenium/Playwright模拟浏览器
- 优点: 免费
- 缺点: 容易被封IP，需要维护

**方案C: RSS/公开接口**
- 淘股吧: 可能有RSS feed
- 豆瓣: 可以爬取公开小组页面
- 优点: 部分平台可用
- 缺点: 不是所有平台都支持
