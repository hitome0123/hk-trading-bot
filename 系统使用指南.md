# 港股实时信号系统 - 完整使用指南

## 🎯 系统架构

```
FutuOpenD (本地行情)
    ↓
Flask API服务 (端口5001) ← Python脚本处理
    ↓
n8n工作流 (端口5678) ← HTTP Request调用
    ↓
钉钉机器人 ← 推送通知
```

**优势**：
- ✅ 不使用`child_process`模块（n8n禁止）
- ✅ 所有逻辑通过HTTP API调用
- ✅ 独立的Python服务，易于调试
- ✅ n8n只负责调度和推送

## 📊 工作流节点说明

### 1. 每5分钟扫描
- **类型**: Schedule Trigger
- **频率**: 每5分钟
- **作用**: 定时触发整个流程

### 2. 获取交易信号 ⭐ 新！
- **类型**: HTTP Request
- **URL**: http://localhost:5001/get_signals
- **作用**: 调用Flask API获取实时信号
- **返回**: JSON格式的信号数据

### 3. 有买入信号？
- **类型**: IF条件判断
- **条件**: `buy_signals > 0`
- **作用**: 只有发现买入信号才继续

### 4. 格式化推送消息
- **类型**: Code节点
- **作用**: 格式化消息（板块/股票/位置/仓位）

### 5. 发送钉钉通知
- **类型**: HTTP Request
- **作用**: 推送到钉钉群

## 🎯 信号条件说明

### 买入信号条件
1. ✅ RSI < 40（超卖）
2. ✅ 布林带位置 < 30%（接近下轨）

- RSI < 30
- 布林带位置 < 20%

### 卖出信号条件
- RSI > 70（超买）
- 布林带位置 > 70%（接近上轨）

## ⚠️ 常见问题

### Q1: API服务启动失败

**检查**:
```bash
# 查看错误日志
cat /tmp/signal_api.log

# 检查端口占用
lsof -i :5001
```

**解决**:
```bash
# 杀死占用端口的进程
pkill -f signal_api.py

# 重新启动
python3 /Users/mantou/hk-trading-bot/signal_api.py &
```

### Q2: n8n工作流报错"Could not find workflow"

**解决**:
```bash
# 重启n8n
pkill -f "n8n start"
n8n start
```

### Q3: 没有收到钉钉推送

**检查清单**:
1. ✅ API服务运行中？ `curl http://localhost:5001/health`
2. ✅ n8n工作流激活？ 查看工作流页面右上角开关
3. ✅ 钉钉webhook配置正确？
4. ✅ 当前是否有买入信号？ `curl http://localhost:5001/get_signals`

### Q4: FutuOpenD连接失败

**检查**:
```bash
# 检查FutuOpenD是否运行
ps aux | grep Futu

# 检查端口11111是否监听
lsof -i :11111
```
