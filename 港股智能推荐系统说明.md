## 系统概述

这是一个**多维度交叉验证的港股智能推荐系统**，整合了：
1. **社交媒体热点监控** - K值趋势分析

3. **交叉验证** - 生成高可信度综合信号

## 核心优势

###  **三维分析模型**

```
社交媒体热点（K值） + 机构资金流向 + 散户资金流向
        ↓                    ↓                ↓
    上升趋势            聪明钱判断          追高警示
        ↓                    ↓                ↓
                   交叉验证整合
                        ↓
                  💎 钻石信号
```

### 🎯 **5种信号类型**

| 信号类型 | 条件 | 强度 | 推荐度 |
|---------|------|------|--------|
| 💎💎 钻石信号 | 热点上升 + 机构买入 | ⭐⭐⭐⭐⭐ | 强烈推荐 |
| 💎⚠️ 博弈信号 | 热点 + 机构 + 散户 | ⭐⭐⭐⭐ | 谨慎参与 |
| 👍 纯机构买入 | 机构买入（无热点） | ⭐⭐⭐ | 稳健投资 |
| 🤔 仅热点信号 | 热点上升（无机构） | ⭐⭐ | 观望为主 |
| ⚠️ 追高警告 | 热点 + 散户买入 | ⭐ | 避免追高 |

## 推送逻辑

### 推送条件（筛选机制）

不是所有信号都推送，只推送高质量信号：

1. **💎💎 钻石信号** - 全部推送（热点+机构，最高优先级）
2. **💎⚠️ 博弈信号** - 全部推送（需要谨慎判断）
3. **👍 纯机构买入** - 只推送**净流入 > 1亿元**的大额信号
4. **🤔 仅热点** - 只推送**K值 > 0.5**的强热点
5. **⚠️ 追高警告** - 全部推送（避雷）

### 推送频率

- **每10分钟检查一次**
- **有新信号才推送**（通过历史记录去重）
- **筛选后平均每次推送2-5条消息**

## 信号整合逻辑

### 交叉验证矩阵

|  | 机构买入 | 散户买入 | 无资金流入 |
|---|---------|---------|-----------|
| **热点上升** | 💎💎 钻石信号 | ⚠️ 追高警告 | 🤔 仅热点 |
| **无热点** | 👍 纯机构买入 | （不推荐） | （忽略） |

### 具体判断规则

```python
if 热点K值 > 0.3 and 机构净流入 > 0 and 散户净流入 <= 0.5亿:
    → 💎💎 钻石信号（强烈推荐）

elif 热点K值 > 0.3 and 机构净流入 > 0 and 散户净流入 > 0.5亿:
    → 💎⚠️ 博弈信号（谨慎参与，看资金占比）

elif 机构净流入 > 1亿 and 热点K值 == None:
    → 👍 纯机构买入（稳健投资）

elif 热点K值 > 0.5 and 机构净流入 == 0:
    → 🤔 仅热点（观望为主）

elif 热点K值 > 0.3 and 散户净流入 > 0.5亿:
    → ⚠️ 追高警告（避免接盘）
```

### 常见问题

**Q1: 为什么有时候没有钻石信号？**

A: 钻石信号需要**热点上升 + 机构买入**同时满足，条件严格。如果：
- K值 < 0.3 → 热点不够强
- 机构净流入 = 0 → 机构没有同步买入

则不会生成钻石信号。

**Q2: 纯机构买入信号可靠吗？**

A: 可靠，但风格不同：
- **钻石信号**：短期爆发力强（有热点催化）
- **纯机构买入**：中长期稳健（机构布局）

建议：钻石信号适合激进，纯机构适合稳健。

**Q3: 如何调整推送频率？**

A: 编辑cron任务：
```bash
crontab -e

# 改为每30分钟
*/30 * * * * /Users/mantou/hk-trading-bot/run_all_signals.sh >> /tmp/all_signals_cron.log 2>&1

*/10 9-16 * * 1-5 /Users/mantou/hk-trading-bot/run_all_signals.sh >> /tmp/all_signals_cron.log 2>&1
```
