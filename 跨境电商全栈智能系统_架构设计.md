#### Node 8: PostgreSQL读取
```yaml
名称: "Load Historical Data"
类型: Postgres
操作: Execute Query
SQL: |
  SELECT
    o.order_id,
    o.product_sku,
    o.quantity,
    p.current_price,
    p.competitor_price,
    i.stock_level
  FROM orders o
  JOIN products p ON o.product_sku = p.sku
  JOIN inventory i ON p.sku = i.sku
  WHERE o.created_at > NOW() - INTERVAL '7 days'
连接池: 5
超时: 10秒
```

#### Node 9: Redis Cache
```yaml
名称: "Check Cache"
类型: Redis
操作: GET

```

### 第3层：条件路由层 (Routing Logic)

#### Node 15: Order Type判断
```yaml
名称: "Route by Order Type"
类型: IF
条件:
  - 新订单 → 触发客服Agent
  - 价格监控 → 触发定价Agent
  - 库存告警 → 触发库存Agent
输出分支: 3个
```

#### Node 16: Priority Switch
```yaml
名称: "Priority Routing"
类型: Switch
规则:
  - Case 1: VIP客户 → 优先级1
  - Case 2: 大额订单(>$1000) → 优先级2
  - Case 3: 普通订单 → 优先级3
  - Default: 低优先级队列
```

#### Node 17: 产品循环
```yaml
名称: "Loop Through Products"
类型: Loop Over Items
输入: {{$json.products}}
批次大小: 10
并行处理: 是
超时: 每项5秒
```

#### Node 18: 阈值检查
```yaml
名称: "Check Thresholds"
类型: IF
条件:
  - 库存 < 安全库存 → 触发补货
  - 价格偏差 > 10% → 需要审批
  - 客户情绪 < 0.3 → 升级人工
```

#### Node 19-20: 结果处理
```yaml
Node 19 - Merge Results:
  类型: Merge
  模式: Wait for all branches
  超时: 60秒

Node 20 - Filter Data:
  类型: Function
  逻辑: 过滤低置信度结果(<0.7)
```

```yaml
名称: "Inventory Management AI Agent"

     Code: |
       from prophet import Prophet

       # 准备数据
       df = pd.DataFrame({
         'ds': dates,
         'y': sales
       })

       # 训练模型
       model = Prophet(
         yearly_seasonality=True,
         weekly_seasonality=True,
         daily_seasonality=False
       )
       model.fit(df)

       # 预测未来30天
       future = model.make_future_dataframe(periods=30)
       forecast = model.predict(future)
```

#### Node 24-29: Agent编排和治理
```yaml
Node 24 - Agent Orchestrator:
  功能: 协调多个Agent的执行
  策略:
    - 并行执行互不依赖的Agent
    - 串行执行有依赖的Agent
    - 超时控制(每个Agent 45秒)

Node 25 - Result Aggregator:
  功能: 聚合所有Agent的输出
  格式化: 统一JSON Schema

Node 26 - Decision Validator:
  功能: 验证AI决策的合理性
  规则:
    - 价格不能低于成本
    - 库存不能为负
    - 客服回复不能包含敏感词

Node 27 - Confidence Scorer:
  功能: 计算综合置信度
  公式: weighted_avg(agent_confidences)
  阈值: 0.75

Node 28 - Human Approval Gate:
  类型: Wait Node
  条件: confidence < 0.75 OR high_risk_action
  超时: 2小时
  通知: Telegram + Email

Node 29 - Fallback Handler:
  功能: 处理Agent失败情况
  策略:
    - 使用规则引擎作为fallback
    - 记录失败原因
    - 触发告警
```

### 第6层：监控层 (Monitoring)

#### Node 35: Error Handler
```yaml
名称: "Centralized Error Handler"
类型: Error Trigger
捕获所有上游错误
处理逻辑:
  - 解析错误类型和原因
  - 分级: ERROR / WARNING / INFO
  - 路由到对应处理流程

错误分类:
  - API错误 → 重试
  - 数据错误 → 记录并跳过
  - Agent错误 → Fallback到规则引擎
  - 系统错误 → 立即告警
```

#### Node 36: Retry Logic
```yaml
名称: "Smart Retry"
类型: Loop + Wait
策略: 指数退避
重试次数: 3
延迟: [2s, 5s, 10s]
条件:
  - 429 Too Many Requests → 等待更长时间
  - 5xx Server Error → 立即重试
  - 4xx Client Error → 不重试，记录
```

#### Node 37: Audit Logger
```yaml
名称: "Audit Trail Logger"
类型: Postgres
表: audit_logs
记录内容:

  - 操作类型
  - 输入数据(脱敏)
  - 输出结果
  - 执行时长
  - 状态码
保留期: 90天
```

#### Node 38: Performance Monitor
```yaml
名称: "Performance Tracker"
类型: Code Node
监控指标:
  - 端到端延迟
```
